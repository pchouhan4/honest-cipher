# Experiment: surface-feature fingerprinting under partial rule leakage

**Date:** March 2026  
**Status:** completed  
**Relates to:** open problem 2 (rule inference attack)

---

## What this experiment asks

Can a machine learning classifier predict a walk's canonical equivalence class using only shallow surface features (token counts, bigram frequencies, term length), given partial knowledge of the private rule set Rs?

This is a weaker question than full rule inference or key recovery. It is a necessary precondition for harder attacks: if canonical class identity leaks into surface statistics, an attacker with partial Rs could bootstrap toward full recovery. If it doesn't, that specific attack path is closed.

---

## What this experiment does not ask

This is not a key recovery attack. It is not a rule inference attack in the LGIP sense — no attempt is made to reconstruct Rs from observations. It does not address algebraic attacks, quantum attacks, or structural analysis of the rewriting system. A negative result here does not imply security under any other attack model.

---

## Setup

**Rules:** 8 public rules (Rp), 222 private rules (Rs), generated by Knuth-Bendix completion of Rp. Rs contains 230 total rules including Rp.

**Dataset:** 512 messages (binary strings 0–511), each encoded as a 24-generator walk. Canonical labels assigned by full Rs-reduction (bounded at 28 steps). This produced 38 unique canonical equivalence classes over the 512 walks — roughly 13–14 walks per class.

**Attacker model:**
1. Observes walks reduced using public rules plus a random subset of private rules (the leaked fraction).
2. Extracts surface features: σ1–σ4 unigram counts, all 16 bigram counts, total length. Features perturbed with Gaussian noise (σ=0.3) to model observation imprecision.
3. Trains a Random Forest classifier (50 trees, max depth 8) on 70% of the dataset.
4. Evaluated on the held-out 30% using balanced accuracy (accounts for class imbalance).

**Leakage fractions tested:** 0%, 1%, 5%, 10%, 25%, 50% of private rules (0, 2, 11, 22, 55, 111 rules).

**Trials:** 50 independent trials per leakage fraction (different random subsets of leaked rules and different train/test splits).

---

## Results

Random-guess baseline: 1/38 ≈ 0.0264

| Leakage | Rules revealed | Mean accuracy | Std | Delta vs. 0% |
|---------|---------------|---------------|-----|--------------|
| 0.0% | 0 | 0.0264 | 0.0030 | — |
| 1.0% | 2 | 0.0264 | 0.0030 | +0.000005 |
| 5.0% | 11 | 0.0264 | 0.0030 | +0.000001 |
| 10.0% | 22 | 0.0264 | 0.0030 | −0.000010 |
| 25.0% | 55 | 0.0264 | 0.0030 | +0.000005 |
| 50.0% | 111 | 0.0264 | 0.0030 | −0.000000 |

The largest observed delta across all conditions is 0.000010 — approximately 333× smaller than the standard deviation (0.003). The accuracy curve is flat within measurement noise.

Withstand capacity (absolute, 5× baseline threshold = 0.132): > 50% — threshold never crossed.  
Withstand capacity (delta-based, +2pp threshold): > 50% — threshold never crossed.

---

## Interpretation

Under this attack model and these parameters, walk surface statistics carry no usable signal about canonical class membership, regardless of how many private rules the attacker knows (up to 50% of Rs). Leaking rules does not improve fingerprinting.

The most likely reason: the rewriting system's equivalence classes are not correlated with the surface-level generator distribution of a walk. Two walks in the same Rs-equivalence class can have very different unigram/bigram profiles — the equivalence is algebraic, not statistical.

---

## Limitations and what this doesn't settle

**Attack model is narrow.** The features used (unigram counts, bigrams, length) are the weakest plausible attacker. An attacker with more computational resources could attempt structural analysis — identifying which rules fire during reduction, reconstructing the critical-pair structure of Rp, or using deeper algebraic features. This experiment says nothing about those approaches.

**Reduction is bounded.** Both the attacker and the canonicalizer use max 28 reduction steps, not full normalization to fixed point. Under full normalization, the equivalence structure changes and results could differ.

**Walk length is short.** 24-generator walks. Longer walks might show different leakage characteristics.

**Rs is dense.** 222 private rules over a 4-generator alphabet is a specific parameter choice. Results for smaller or sparser Rs may differ.

**Negative result, not a proof.** This is empirical evidence against one attack. It is not a proof of hardness, not a reduction, and does not address open problem 1 (average-case hardness of LGIP). A more sophisticated attacker might succeed where this one fails.

## What remains open

Open problem 2 (rule inference attack) is partially addressed here — the surface-feature path appears closed under this model. What remains:

- Direct algebraic reconstruction of Rs from known (plaintext, ciphertext) pairs
- Key recovery from known-plaintext pairs via permutation table analysis
- Statistical attacks using deeper features (reduction paths, critical pair fingerprints)
- Attacks under longer walks and larger rule sets

A break via any of these would be a significant result. If you find one, open an issue.
